# GAS Web App 部署指南

## 步驟 1：建立 GAS 專案

1. 前往 [Google Apps Script](https://script.google.com)
2. 點擊「新專案」
3. 將 `Code.gs` 的內容貼入編輯器
4. 專案命名為「大安籃球聯盟 API」

## 步驟 2：填入 Sheets 設定

在 `Code.gs` 頂部的 `CONFIG` 區塊中填入：

```javascript
SHEETS: {
  MAIN: '從 Google Sheets URL 中取得的 ID'
  // URL: https://docs.google.com/spreadsheets/d/{THIS_PART}/edit
}
```

並將 `TABS` 中的工作表名稱改為你實際的 tab 名稱。

## 步驟 3：設定 Sheets 權限

GAS 需要存取 Sheets 的權限：
1. 在 GAS 編輯器中，執行任一函式（如 `testSingle`）
2. 系統會要求授權 → 同意即可

## 步驟 4：測試

1. 在 GAS 編輯器中選擇 `testDoGet` 函式
2. 點擊「執行」
3. 查看 Logger 輸出（檢視 → 記錄）確認資料正確

## 步驟 5：部署 Web App

1. 點擊「部署」→「新增部署」
2. 類型選擇「Web 應用程式」
3. 設定：
   - 說明：`v1.0`
   - 執行身分：**我**
   - 誰可以存取：**所有人**
4. 點擊「部署」
5. 複製產生的 URL（格式：`https://script.google.com/macros/s/xxxxx/exec`）

## 步驟 6：更新前端

在 `js/api.js` 中將 `GAS_URL` 改為部署 URL：

```javascript
GAS_URL: 'https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec',
```

## 步驟 7：驗證

在瀏覽器中測試各 API：
- `{GAS_URL}?type=roster`
- `{GAS_URL}?type=schedule`
- `{GAS_URL}?type=standings`
- `{GAS_URL}?type=stats`
- `{GAS_URL}?type=dragon`
- `{GAS_URL}?type=rotation`
- `{GAS_URL}?type=hof`
- `{GAS_URL}?type=home`

## 更新部署

修改 GAS 程式碼後需要重新部署：
1. 點擊「部署」→「管理部署」
2. 點擊鉛筆圖示編輯
3. 版本選「新版本」
4. 點擊「部署」

> 注意：每次更新程式碼都必須建立新版本，否則前端仍會使用舊版。

## 常見問題

**Q: GAS 回應很慢？**
A: GAS 冷啟動可能需要 3-5 秒。前端 api.js 已將超時設為 15 秒。

**Q: CORS 錯誤？**
A: 確認部署時「誰可以存取」設為「所有人」。

**Q: 回傳 HTML 而非 JSON？**
A: 可能是程式碼有錯誤。在 GAS 編輯器中執行 `testSingle` 檢查。

**Q: 想保留靜態 JSON 作為備份？**
A: 前端 api.js 已內建 fallback 機制。GAS 請求失敗時會自動回退至 `./data/` 目錄的靜態 JSON。
